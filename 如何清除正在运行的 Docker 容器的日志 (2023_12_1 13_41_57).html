如何清除正在运行的 Docker 容器的日志

长时间运行的 Docker 容器会迅速积累大量的日志行。这些会消耗存储容量并降低访问和过滤数据时的性能。虽然 Docker 包含用于查看日志的集成工具，但没有用于清理它们的内置机制。

在本文中，您将学习如何清除正在运行的 Docker 容器的日志，而无需重新启动或替换它们。您还将看到一些在保留旧数据的同时更有效地处理大型日志的技术。

理解问题

Docker 从容器前台进程的标准输出和错误流中收集日志。 docker logs 命令用于检索这些日志，但它不允许您删除旧行。

Docker 支持许多不同的日志记录驱动程序，因此无法提供标准化的清理机制。本文重点介绍标准的 json-file 日志记录驱动程序，其中日志行存储在容器主机文件系统上的 JSON 文件中。如果您需要清理流式传输到远程主机的日志，请参阅存储驱动程序的文档。

json-file 驱动程序创建的日志文件存储在 /var/lib/docker/containers 目录下。每个容器都有自己的日志文件，该文件在其整个生命周期中使用。默认情况下没有配置日志轮换。

简单地删除日志文件并不是一个有效的解决方案。 Docker 期望该文件持续可用，并且如果它被删除不会自动重新创建它。更安全的做法是清除现有日志的内容以避免影响任何当前的写入。

发现日志文件的路径

首先找到目标容器日志文件的路径。您可以通过运行以下命令检索名为 my-app 的容器的日志文件路径：

$ docker inspect --format='{{.LogPath}}' my-app
/var/lib/containers/1380d936...-json.log

清除日志文件

您可以通过将空字符串回显到其内容中来清除日志的内容而不删除它。该文件将归 root 所有，因此您需要在 root shell 中执行此操作。以下命令将为您清空日志文件：

$ sudo sh -c 'echo "" > $(docker inspect --format="{{.LogPath}}" my-app)'

Shell 插值用于动态检索 my-app 容器的日志文件路径。您可以手动替换之前检索到的路径。

当您运行 docker logs my-app 时，您现在将看到空输出，除非容器在此期间恢复写入行。

限制日志输出

docker logs 命令接受一些参数，这些参数可用于限制噪声日志的输出。这些有助于减少清理日志的需要，同时防止您的终端在每次访问文件时被旧行淹没。

  • --tail – 此标志指示 Docker 仅发出特定数量的旧日志行，从最近的开始。
  • --until 和 --since – 这些标志将输出限制为在特定时间段内发出的日志行。当您知道事件发生的大致时间范围时，它们会很有用。

这里有一些例子：

显示最近的 100 行日志

docker logs my-app --tail 100

显示过去一小时内写入的日志行

docker logs my-app --since 1h

最多显示过去一小时内写入的 100 行日志

docker logs my-app --tail 100 --since 1h

设置日志循环

必须定期以这种方式清理日志文件通常表明您的应用程序日志要么过于冗长，要么就是有太多活动无法存储在一个文件中。

许多 Docker 日志记录驱动程序，包括 json-file，都具有可选的日志轮换支持，您可以为 Docker 守护进程全局启用或在每个容器的基础上启用。

守护进程设置在 /etc/docker/daemon.json 中配置。这是一个示例，一旦容器日志达到 8MB，就会轮换日志。最多可随时保留五个文件，旧文件会在新轮换发生时自动删除。

{
    "log-opts": {
        "max-size": "8m",
        "max-file": "5"
    }
}

重新启动 Docker 守护进程以应用更改：

$ sudo service docker restart

守护进程级轮换适用于所有新创建的容器。更改不会影响主机上已存在的任何容器。

可以使用 --log-opts 标志为单个容器配置轮换。这些将覆盖您的默认 Docker 守护程序设置。

docker run --name app \
    --log-driver json-file \
    --log-opts max-size=8M \
    --log-opts max-file=5 \
    app-image:latest

概括

当繁忙的应用程序写入大量输出时，Docker 容器日志会变得嘈杂。访问详细日志有助于调试，但它们会占用存储空间，使筛选数据变得更加困难，并可能降低性能。

您可以通过使用 shell 命令清空日志文件的内容来按需清理日志文件。然而，更好的做法是设置日志轮换，以便在文件达到特定大小时自动发生。 docker logs 命令还支持可以帮助在访问点驯服笨重日志的标志。
